# 1. Terminology

The following terms may be familiar to those with experience in user-interface design; however, many intentionally differ from their traditional meanings and are reintroduced to refine, clarify, and unify the language used when discussing the CatalystUI Model.

## Definitions

### Process

A *process* is something that accepts an *input*, performs a *transformation*, and produces an *output*.

### System

A *system* is an organized set of processes that defines how they interact and the order in which they are executed.

### Interface

An *interface* is a mechanism which allows two differing systems to interact.

### Information System

An *information system* is a specialized system that works with “persistent data,” or data that remains available beyond a single process and allows a user to create, read, update, and delete it through an interface over time.

### Computer

A *computer* is a form of information system which exclusively works with “digital data,” or data that can be represented in a binary format and processed by a machine.

### User

A *user* is a person who can interact with an information system.

## Notes for Consistency

### Information vs. Data

Throughout this text, “data” and “information” are used interchangeably. Although the terms can be considered distinct, the Model treats them as equivalent for the sake of conciseness.

### Specialized Forms of Data

The CatalystUI Model will commonly use the following terms to represent three specialized forms of data:

- *Digital data*; any form of information which exists in binary (1s and 0s),
- *Interaction data*; any form of information that expresses the intent behind an action, and
- *Perceivable data*; any form of information which a user can perceive.
